Example description

  
   	USB CDC
   	
DESCRIPTION
===========
	 The usb device example emulates a COM port in a Windows HOST using the 
	Comunication Device Class (CDC). The USB is recognized as a standard COM 
	port and creates an echo aplication. This means that everything that goes 
	out from the terminal returns to the terminal.
	
	Notes:
   After the micro USB and the PC are connected, the PC will request a driver, 
  point to the Freescale_CDC_Driver_kinetis.inf file located in the usb_device 
  project folder. 
   Re-enumeration for this demo is not supported, for TWR-K21D50M does not have 
  circuit to detect VBUS connection.

CONFIGURATION
=============
  The Workspace contains example with few configurations:

  RAM_128KB 											: The example is loaded into the internal RAM.
  FLASH_1M_PFLASH									: The example is flashed into the embedded flash.
  FLASH_512KB_PFLASH_512KB_DFLAHS	: The example is flashed into the embedded flash.
  
COMPATIBILITY
=============

   The example project is compatible with TWR-K21F evaluation board.
  By default the project is configured to use on board PE micro debugger.

GETTING STARTED
===============

  1) Start the IAR Embedded Workbench for ARM.

  2) Connect the on board OpenSDA debugger

  3) Press Ctrl+D or use Download and Debug button to start a debug session
